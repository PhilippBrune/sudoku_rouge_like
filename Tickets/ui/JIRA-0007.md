# JIRA-0007 — Scene Setup (MainMenu + Prototype)

**Created:** 2026-03-07

## Summary

All `.unity` scene files were lost (likely deleted with the mirror tree cleanup). The project needs two scenes recreated: **MainMenu** (build index 0) and **Prototype** (build index 1). Both scenes are almost entirely built at runtime by blueprint builder scripts — only a handful of GameObjects with scripts need to be placed manually.

## Requirements

### R1 — Create MainMenu scene
- Scene saved as `Assets/Scenes/MainMenu.unity`
- Contains one GameObject with `MainMenuBlueprintBuilder` + `MainMenuController` components
- `MainMenuBlueprintBuilder.autoBuildOnStart = true` (default)
- Everything else (Canvas, Camera, EventSystem, all panels) is created at runtime by the builder

### R2 — Create Prototype scene
- Scene saved as `Assets/Scenes/Prototype.unity`
- Contains a "Bootstrap" GameObject with `GameBootstrap` + `InRunUiBlueprintBuilder` components
- Contains a "RunMap" GameObject with `RunMapController` component
- `InRunUiBlueprintBuilder.runMapController` serialized reference points to the RunMap object
- Everything else (Canvas, Camera, EventSystem, UI panels) is created at runtime

### R3 — Build Settings
- MainMenu at build index 0
- Prototype at build index 1
- Both scenes added via File → Build Profiles → Scene List
