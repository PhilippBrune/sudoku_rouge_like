# JIRA-0009 — Boss Mechanics Implementation

**Created:** 2026-03-08

## Summary

Implement 8 boss modifier mechanics that alter Sudoku gameplay rules during boss encounters. Players choose one of two randomly offered mechanics before facing the boss. Mechanics that have never been played show as "???" until encountered.

## Requirements

### Fog of War
- [x] R1 — Implement Fog of War modifier — DONE: FogOfWarRule + GenerateFog hides non-given cells, RevealAdjacentFog on correct placement, fog blocks input, clears on completion

### Arrow Sum
- [x] R2 — Implement Arrow Sum modifier — DONE: ArrowSumRule validates path sum = circle digit, GenerateArrows places 2-3 valid arrows on solved board

### German Whispers
- [x] R3 — Implement German Whispers modifier — DONE: GermanWhispersRule validates ≥5 difference, green line overlay rendering

### Dutch Whispers
- [x] R4 — Implement Dutch Whispers modifier — DONE: DutchWhispersRule validates ≥4 difference, orange line overlay rendering

### Parity Lines
- [x] R5 — Implement Parity Lines modifier — DONE: ParityLinesRule validates alternating odd/even, blue line overlay rendering

### Renban Lines
- [x] R6 — Implement Renban Lines modifier — DONE: RenbanLinesRule validates consecutive set, pink line overlay rendering

### Killer Cages
- [x] R7 — Implement Killer Cages modifier — DONE: KillerCageRule validates sum + no repeats, cage border tint + sum label rendering

### Kropki Dots
- [x] R8 — Implement Kropki Dots modifier — DONE: DifferenceKropkiRule (white dots, consecutive) + RatioKropkiRule (black dots, 1:2 ratio), dot overlay rendering
