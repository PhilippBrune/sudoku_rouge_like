# JIRA-0014 — Regression Pass: Menu Buttons, Garden Tiles, Boss Layer, Killer Cage, XP Loop

**Created:** 2026-03-08

## Summary

Follow-up on JIRA-0013. All items below were marked done but are still failing in-game. This ticket tracks the remaining regressions and clarifies the correct expected behaviour.

---

## Requirements

### Main Menu — Button Text (regression from JIRA-0013 R1)

- [x] R1 — Main menu buttons have no visible text. The SetAsLastSibling / Shadow fix from JIRA-0013 R1 did not resolve the issue in the running build.
  - **Expected:** Buttons look exactly like `docs/icons/main_menue.png` — text is readable on top of the sliced background image.
  - **Fix approach:** Verify the label GameObject is created *after* the background Image, or call `label.transform.SetAsLastSibling()` explicitly. Confirm the Canvas render mode and sorting order allow the label to appear in front.

---

### Garden Path Overview — Tile Info Labels (regression from JIRA-0013 R2)

- [x] R2 — Path tiles do not show board **size** (top-left corner) or **difficulty** (bottom-right corner). This is absent on ALL node types, including Rest, Shop, and Relic tiles.
  - **Expected:** Every tile on the path overview shows the board size (e.g. "6×6") in the top-left corner and the difficulty rating (e.g. "★★☆") in the bottom-right corner. Rest, Shop, and Relic tiles should also display these values based on what the player would face next at that depth.
  - **Fix approach:** The size/difficulty overlay must be applied to every node button, not just Puzzle nodes. Check `CreatePathNodeButton` — size/difficulty label creation must happen outside any `if (nodeType == Puzzle)` guard.

---

### Garden Path Overview — Skip Bug (regression from JIRA-0013 R3)

- [x] R3 — After clicking **Skip** on the reward panel the run becomes stuck: the path overview reappears but selecting a new tile and pressing it does not start a new puzzle.
  - **Expected:** Skip dismisses the reward panel, returns to the path overview, and the player can select and enter the next tile normally.
  - **Fix approach:** Check that `HideRewardPanel` properly resets all "reward pending" state flags (e.g. `_pendingReward`, `_awaitingReward`) so `OnPathNodeClicked` is not blocked. The Skip handler must call the same cleanup path as accepting a reward.

### Options — Highlight Errors Toggle (regression from JIRA-0013 R7)

- [x] R4 — Toggling **Highlight Errors** in the Options panel has no effect during a puzzle. Errors remain coloured even when the toggle is OFF.
  - **Expected:** When the toggle is OFF, incorrect cell placements show no extra colour (same appearance as an unset cell or a correct placement). When toggled ON again, error highlighting resumes immediately.
  - **Fix approach:** `SetHighlightConflictsLive()` must trigger a full redraw of all cell backgrounds. Confirm `_highlightConflicts` is read inside `GetCellBackgroundColor` (or equivalent) on every paint call, not cached at puzzle-load time only.

### Boss Mechanics — Puzzle Layer Order (regression from JIRA-0013 R8)

- [x] R5 — The fix in JIRA-0013 R8 moved lines behind everything; they are now invisible. The correct stacking order was not achieved.
  - **Expected layer order (back → front):**
    1. **Grid** — background cell rectangles and region borders.
    2. **Lines & icons** — modifier overlays (whisper lines, arrow paths, Kropki dots, cage borders, fog tiles).
    3. **Numbers** — given digits and player-entered digits, always on top.
  - **Fix approach:** Create three explicit sibling roots under the sudoku canvas: `GridRoot`, `OverlayRoot`, `NumberRoot`. Place cell backgrounds in `GridRoot`, all modifier visuals in `OverlayRoot`, and all digit labels in `NumberRoot`. Sibling order (0=back, 2=front) enforces the layering.

### Boss Mechanics — Killer Cage (regression from JIRA-0013 R9)

- [x] R6 — Killer Cage has three separate sub-issues:
  1. **Cage border not visible** — cage outlines are not rendered on the grid during gameplay. They should always be visible (not only on hover or selection).
  2. **Cage sum number missing / wrong position** — the sum label is not shown, or is not positioned in the **top-left corner** of the cage's top-left cell.
  3. **Sum number hard to read** — use **red font** for the cage sum digit to contrast against the grid.
  - **Expected:** Each cage is outlined at all times with a clearly visible border. The sum appears as a small red number in the top-left corner of the cage's first (top-left) cell. The board remains fully interactive while the cage is displayed; the **board border turns red** to signal the active Killer Cage modifier.
  - **Fix approach:**
    - Ensure cage border GameObjects are created at puzzle-load time and stay active.
    - Place the sum label as a child of the top-left cell's `NumberRoot` layer, anchored top-left, small font size (~9–11), color `Color.red`.
    - Set the outer `GridPanel` or board frame image color to red (`new Color(0.9f, 0.1f, 0.1f, 1f)`) when the Killer Cage modifier is active.

### XP & Class Progression — Full Loop (regression from JIRA-0012 R1 and JIRA-0013 R10)

- [x] R7 — XP is still not correctly reflected in the **Select Class** screen after a game-over.
  - **Expected full loop:**
    1. A class starts at Level 1 with 0 XP.
    2. Completing a run awards XP (shown on the Game Over screen).
    3. That exact XP value is added to the class that was played.
    4. If the accumulated XP meets or exceeds the level threshold, the class levels up (and may level up multiple times).
    5. The **Select Class** screen shows the current level, a progress bar (or numeric display) of XP toward the next level, and the total XP earned.
  - **Fix approach:**
    - Verify `ClassGardenProgressionService.ApplyRun` is called with the correct class ID (the one that was active during the run, not the currently selected class).
    - Verify the persisted profile data is saved to disk *before* the Select Class screen loads.
    - In the Select Class UI builder, read `GetClassLevel(classId)` and `GetClassXp(classId)` from `ClassGardenProgressionService` and render them for every class card, updating when a class is selected.
    - Add a `currentXp / xpToNextLevel` progress indicator beneath each class name.
